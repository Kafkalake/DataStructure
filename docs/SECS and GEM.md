# Overall
SECS와 GEM은 반도체, 디스플레이, 태양광 등의 첨단 제조 공장에서 <br>
설비(Equipment)와 공장을 제어하는 상위 시스템(Host/MES)이 대화하기 위해 사용하는 국제 표준 통신 프로토콜입니다. <br>
국제반도체장비재료협회인 SEMI에서 제정했습니다.<br>

- GEM (행동 지침): "온도가 180도를 넘었으니 Host에게 경고를 보내야겠다."
- SECS-II (메시지 변환): "경고 메시지는 S5F1 포맷으로 포장해야지."
- HSMS (전송): "포장된 메시지를 TCP/IP 랜선을 통해 Host 서버로 쏜다!"

이러한 SECS/GEM 표준 덕분에, 삼성전자나 SK하이닉스 같은 반도체 공장에서는 <br>
전 세계 수백 개의 서로 다른 장비 업체(ASML, Applied Materials, TEL 등)에서 장비를 사 오더라도, <br>
별도의 커스텀 개발 없이 공장 전체를 하나의 시스템(MES)으로 묶어서 100% 무인 자동화 공장을 구축합니다.

## SECS (Semiconductor Equipment Communication Standard)
SECS는 데이터를 '어떻게 운반하고, 어떤 문법으로 작성할 것인가'를 정의하는 규약입니다.

### 1. HSMS (High-Speed SECS Message Services) / 구 SECS-I
- 역할: 데이터의 운반 

### 2. SECS-II (SEMI E5)
- 역할: 보내는 메시지의 형식을 규정합니다. 
- 메시지를 SxFy(Stream(대분류) + Function(소분류)) 형태로 정의합니다.

예시)
S1F1 (Host가 설비에게): "너 통신 가능한 상태니? (Are you there?)" ->
S1F2 (설비가 Host에게): "응, 나 통신 가능해. (I am here)" ->
S5F1: "알람 발생했어!"

## GEM (Generic Equipment Model / SEMI E30)
SECS-II가 단어와 문법을 정해두긴 했지만, "어떤 상황에서 어떤 단어를 써야 하는지"에 대한 업무 매뉴얼이 없으면 중구난방이 됩니다.
A사 설비와 B사 설비가 똑같은 상황에서 서로 다른 메시지를 보낼 수 있기 때문입니다. 
이러한 설비의 '동작 시나리오와 상태'를 통일한 업무 규정집이 바로 GEM입니다.

- 역할: 공통 동작 모델 및 비즈니스 로직

### GEM이 규정하는 핵심 기능들

- 통신 상태 관리 (Control State): 설비가 Host와 온라인 상태인지, 오프라인 상태인지, Host가 설비를 원격 제어할 권한이 있는지를 관리합니다.
- 데이터 수집 (Data Collection): Host가 "설비야, 압력과 온도 센서 데이터를 1초마다 보고해(Trace)"라거나, "특정 공정(Step)이 끝날 때마다 결과 값을 묶어서 보고해(Report)"라고 명령할 수 있는 체계를 제공합니다.
- 알람 관리 (Alarm Management): 설비에 문제가 생겼을 때 표준화된 형식으로 알람을 보고합니다.
- 스풀링 (Spooling): 통신이 잠시 끊어졌을 때 설비가 데이터를 버리지 않고 자체 저장해 두었다가, 연결이 복구되면 밀린 데이터를 쏟아내는 기능입니다.
